Buddy Abroad

O tema do projecto foi escolhido de modo a conjugar o desenvolvimento do nosso trabalho final de curso com a unidade curricular "Programação Web".

Buddy Abroad App

A Buddy Abroad é solução mobile, constituída por duas aplicações(A de turistas e a de buddies/Guias) que pretende conciliar os novos padrões de consumo consequentes da nova realidade imposta pela COVID-19. Com a ausência no mercado de uma aplicação de viagens capaz de ligar o desejo que os turistas têm por experiências genuínas, únicas e personalizadas, com residentes locais, disponíveis e aptos para satisfazer este desejo.

Vídeo

Visão Single-page application

Como a nossa visão inicial para o projecto era uma single-page application decidimos incorporar apenas para efeitos lúdicos, sem qualquer peso na avaliaçao, uma visão single-page application. Podem aceder a esta visão carregando no botão abaixo sendo possível sair dessa mesma visão carregando no botão VOLTAR na barra de navegação no canto superior direito.

Técnicas usadas

As técnicas mais usadas durante este trabalho foram o CSS Flexbox e CSS Grid, devido à sua utilidade em transformar o website responsivo e de permitir uma disposição esteticamente agrádavel dos elementos.

Layouts

Sendo o âmbito do nosso projecto desenvolver uma landing page para publicitar as nossas aplicações móveis, optamos por um column layout. Este layout é o mais utilizado entre os diversos layouts usados em landing pages de aplicações móveis.

No inicio do desenvolvimento deste projecto tinhamos em mente desenvolver uma single-page application mas visto que não é permitido apenas uma única página nesta entrega, optamos por transformar o que iriam ser as diferentes secções(#sections de um .html) em páginas separadas(diversos ficheiros .html).

Layouts Usámos o CSS grid e CSS flexbox para construção de layouts. Grid utilizado no index.html para o display do texto responsivo por cima da imagem de background. Flex utilizado em quase todas as páginas para garantir a responsividade da informação.

Etiquetas HTML5 main, header, article, footer, figure... Main e Header em todas as páginas, footer, figure e article usados apenas quando fazem sentido.
Animações Foram feitas animações usando keyframes. Na pagina de Avaliação foram usadas para fazer a troca cíclica de nome de guide para tourist e mudança de cor do titulo buddy abroad na mesma página.
